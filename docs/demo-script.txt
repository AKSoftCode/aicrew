/dev — 9 phases (features, refactors, multi-system work)
  0 Intake       → acceptance criteria + pipeline confirmed
  1 Research     → Scout pass; root cause OR key files classified
  2 Brainstorm   → 3 alternatives; user picks approach
  3 Design       → architect spec confirmed — no code yet
  4 Implement    → TDD strict: RED → GREEN → REFACTOR
  5 Tests        → test engineer PASS; new code coverage ≥ 80%
  6 Security     → security-reviewer PASS on changed files only
  7 Audit        → project /audit PASS (if included at intake)
  8 Cloud/Infra  → auto on schema, deps, deploy changes
  9 Conclude     → memory saved, diff --stat, commit msg ready

/fix — 5 phases (known bugs — fast path, fixed pipeline)
  1 Bug Analysis → confirmed root cause BEFORE any code
  2 Implement    → failing repro test → fix → suite green
  3 Tests        → test engineer PASS; coverage ≥ 80%
  4 Security     → security-reviewer PASS on changed files
  5 Conclude     → summary, diff --stat, commit message ready

/quick — Scout → Act (small scoped tasks)
  Scout → graph-first discovery (haiku/mini) → SCOUT: schema block
  Act   → surgical change with Scout context only (sonnet/opus)
  Escalate to /fix for bugs; to /dev for design spec or infra.

Token economics (illustrative discovery cost):
  repo-wide grep scan     ~80,000 tokens
  graph MCP query (Scout) ~500 tokens  (~99% savings)
