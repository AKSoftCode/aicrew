/dev  — 9 phases
  Intake → Research → Brainstorm → Design → Implement (TDD)
  → Tests → Security → Audit → Conclude
  Use: features, refactors, multi-system changes

/fix  — 5 phases
  Bug Analysis → Implement (TDD) → Tests → Security → Conclude
  Use: known bugs — TDD + security, no ceremony

/quick — Scout → Act
  Graph-first discovery → surgical implementation
  Use: rename, tweak, small scoped additions
